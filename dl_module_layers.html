Bài 7: Kiến trúc Lego

Lập trình Mạng Nơ-ron hướng đối tượng (OOP) với Khái niệm Module

1. Từ Nơ-ron lẻ tẻ đến Khối Module

Trong Deep Learning, sự trừu tượng hóa (Abstraction) chia làm 4 cấp độ. Giống như chơi Lego:

🧱

1. Nơ-ron

Một cục gạch Lego 1x1. Thực hiện tính: \(wx + b\)

📏

2. Layer (Lớp)

Ghép nhiều nơ-ron thành 1 thanh ngang. (VD: Dense)

📦

3. Module (Khối)

Ghép nhiều Layers lại thành một khối có chức năng riêng.

🏰

4. Model (Mạng)

Ghép các Module lại thành một tòa lâu đài hoàn chỉnh.

Tại sao cần Module? Trong kiến trúc thực tế như ResNet hay Transformer, người ta phát hiện ra các mẫu (patterns) được lặp đi lặp lại. Việc viết code tạo ra một "Khối" (Module), sau đó tái sử dụng (nhân bản) khối đó lên 10 lần, 100 lần sẽ giúp code ngắn gọn, dễ debug và cực kỳ linh hoạt.

2. Giải phẫu một Module (Subclassing)

Trong TensorFlow, một Module được tạo ra bằng cách kế thừa lớp tf.keras.Model (hoặc tf.keras.layers.Layer). Một Module bắt buộc phải có 2 hàm "thần thánh":

  • __init__: Nơi bạn gom đồ nghề. Bạn khai báo các Layers, các trọng số cần thiết ở đây. (Giai đoạn chuẩn bị vật tư).
  • call: Nơi bạn viết hướng dẫn lắp ráp. Bạn định nghĩa cách dữ liệu X chảy qua các "đồ nghề" đã chuẩn bị ở trên. (Giai đoạn thi công: Forward Propagation).
import tensorflow as tf

class MyCustomMLP(tf.keras.Model): # Kế thừa class Model
    def __init__(self):
        super().__init__() # Bắt buộc: Gọi hàm init của lớp cha
        # 1. Gom đồ nghề
        self.hidden = tf.keras.layers.Dense(256, activation=tf.nn.relu)
        self.out = tf.keras.layers.Dense(10)

    def call(self, X):
        # 2. Hướng dẫn dòng chảy dữ liệu (Forward Pass)
        hidden_output = self.hidden(X)
        final_output = self.out(hidden_output)
        return final_output

# Sử dụng:
net = MyCustomMLP()
# Gọi net(X) thực chất là Python đang âm thầm gọi hàm net.call(X)

3. Bản chất của `Sequential` và Lồng ghép (Nesting)

Hàm Sequential mà chúng ta dùng suốt từ Bài 1 thực chất là một Module được TensorFlow viết sẵn. Nó nhận vào một danh sách các Module, và trong hàm call của nó chỉ đơn giản là một vòng lặp for chạy dữ liệu qua từng Module một.

Sức mạnh lớn nhất của lập trình hướng đối tượng là Lồng ghép (Nesting). Một Module có thể chứa các Module khác bên trong nó!

# Bạn có thể tạo ra những con "Quái vật Chimera" như thế này:
chimera = tf.keras.Sequential([
    MyCustomMLP(),                     # Nhét Module tự chế vào Sequential
    tf.keras.layers.Dense(20),         # Nhét Layer bình thường vào
    AnotherCustomModule()              # Nhét thêm một Module khác
])

4. Mạng Nơ-ron hay một Chương trình Máy tính?

Đây là điểm thú vị nhất trong tài liệu của bạn (Đoạn FixedHiddenMLP). Ngày xưa, mạng Nơ-ron chỉ là các phép nhân ma trận cứng nhắc. Ngày nay, trong hàm call(), bạn có thể viết BẤT KỲ CODE PYTHON NÀO: lệnh if/else, vòng lặp while, tính toán số học tùy ý.

Khuôn khổ TensorFlow sẽ tự động theo dõi các phép toán này (AutoGraph) để tính đạo hàm ngược (Backpropagation) một cách thần kỳ!

🖥️ Trình giả lập: Custom Forward Pass với lệnh `while`

Giả lập đoạn code trong sách: Dữ liệu bị nhân lên, cộng thêm 1, sau đó chạy qua vòng lặp `while`. Nếu tổng trị tuyệt đối > 1, chia đôi dữ liệu liên tục cho đến khi thỏa mãn.
