Lưu "Save Game" và Mở khóa tốc độ song song

1. Lưu trữ: Nút "Save Game" của AI

Huấn luyện (Training) một mô hình Deep Learning giống như chơi một tựa game nhập vai cực khó mất hàng tuần liền. Nếu bạn vô tình đá phải phích cắm điện của máy chủ mà chưa lưu lại, toàn bộ "Trí thông minh" (Weights) sẽ tan thành mây khói.

Bí quyết thực chiến: Chúng ta thường KHÔNG lưu toàn bộ mô hình (bao gồm cả code kiến trúc mạng). Lý do là vì code có thể thay đổi, và mã nguồn Python lưu dạng nhị phân rất dễ bị lỗi tương thích.

Thay vào đó, ta viết code tạo ra "cái vỏ" (kiến trúc), và chỉ LƯU LẠI BỘ TRỌNG SỐ (Weights/Parameters) ra file tĩnh. Khi cần dùng, ta tải bộ trọng số đó lắp vào cái vỏ. 
import tensorflow as tf
import numpy as np

# --- 1. LƯU MẢNG DỮ LIỆU THƯỜNG (TENSORS) ---
x = tf.range(4)
np.save('x-file.npy', x)               # Lưu ra file
x2 = np.load('x-file.npy', allow_pickle=True) # Tải lại vào RAM

# --- 2. LƯU & TẢI TRỌNG SỐ CỦA MẠNG NƠ-RON ---
net = MyCustomMLP() 
net(tf.random.uniform((2, 20))) # Chạy mồi 1 lần để Lazy Init tạo Weights

# Lưu game! (Lưu trạng thái weights hiện tại)
net.save_weights('my_model_weights.h5')

# Ở một máy tính khác, hoặc một ngày khác:
clone_net = MyCustomMLP() # Khởi tạo lại cái vỏ
clone_net(tf.random.uniform((2, 20))) # Chạy mồi

# Tải file Save (Lắp não vào vỏ)
clone_net.load_weights('my_model_weights.h5')

2. Tại sao AI lại cuồng GPU (Card Đồ Họa)?

Bạn có bao giờ thắc mắc tại sao NVIDIA lại trở thành công ty nghìn tỷ đô? Bí mật nằm ở kiến trúc phần cứng:

  • CPU (Giáo sư Toán học): Có khoảng 8 đến 24 nhân (cores) siêu mạnh, siêu thông minh. Nó giải quyết các bài toán logic phức tạp (if/else), chạy hệ điều hành rất tuyệt vời. Nhưng nó chỉ làm được từng việc một.
  • GPU (Hàng ngàn học sinh cấp 1): Có hàng nghìn nhân nhỏ (VD: RTX 4090 có hơn 16,000 nhân CUDA). Mỗi nhân khá "ngốc", chỉ biết làm phép nhân cộng đơn giản. NHƯNG chúng có thể làm việc đó Cùng Một Lúc (Xử lý song song).

Vì Mạng Nơ-ron về bản chất là hàng triệu phép Nhân Ma Trận (Matrix Multiplication), việc giao nó cho "Đội quân học sinh" GPU sẽ nhanh hơn hàng trăm lần so với vị Giáo sư CPU!

# Kiểm tra xem máy có GPU NVIDIA không?
print(tf.config.list_physical_devices('GPU'))

# Bắt buộc tạo Tensor trên một thiết bị cụ thể
with tf.device('/CPU:0'):
    X = tf.ones((10000, 10000)) # Lưu trên RAM thường

with tf.device('/GPU:0'):
    Y = tf.random.uniform((10000, 10000)) # Lưu trên VRAM của Card màn hình

3. Nghịch lý tốc độ: Nút thắt cổ chai PCI-e

Tài liệu của bạn ở phần Side Notes (Lưu ý bên lề) chỉ ra một lỗi kinh điển của người mới: Truyền dữ liệu qua lại giữa CPU và GPU quá nhiều.

Quy tắc thép: GPU tính toán cực kỳ nhanh, nhưng con đường vận chuyển dữ liệu từ RAM máy tính sang VRAM của GPU (qua khe cắm PCI-Express trên Mainboard) lại rất chậm!

Nếu bạn liên tục tính 1 phép toán nhỏ trên GPU, rồi copy kết quả về CPU để in ra màn hình (`print(loss.numpy())`), bạn đang làm hệ thống kẹt xe cứng ngắc.

🚀 Trình Mô Phỏng: Data Transfer vs Computation

Hãy thử chuyển một mảng dữ liệu (Batch) từ RAM hệ thống sang Card Đồ họa, sau đó nhấn tính toán để thấy sự chênh lệch thời gian.

Hệ thống sẵn sàng...
Data
🖥️ CPU (System RAM)
Chứa dữ liệu Dataset lớn tải từ ổ cứng.
Cáp PCI-Express (Chậm)
🎮 GPU (VRAM)
16,000 CUDA Cores đang chờ lệnh.

Bài học: Đừng chuyển từng tấm ảnh qua GPU. Hãy gom chúng thành 1 Batch lớn (ví dụ 64 ảnh) chuyển qua 1 lần, rồi để GPU tính toán toàn bộ. Hạn chế dùng hàm .numpy() trong vòng lặp Training.

4. Sức mạnh Đa GPU (Multi-GPU)

Khi mô hình quá lớn (như LLM), 1 card GPU là không đủ. Chúng ta cắm 4, 8 chiếc GPU vào cùng một máy. TensorFlow cung cấp Distribution Strategies để quản lý việc này một cách kỳ diệu.

import tensorflow as tf

# Khai báo chiến thuật: Nhân bản mô hình ra TẤT CẢ các GPU hiện có trên máy
strategy = tf.distribute.MirroredStrategy()

print(f"Số lượng GPU đang tham gia: {strategy.num_replicas_in_sync}")

# Đặt code tạo mô hình trong "Scope" của Strategy
with strategy.scope():
    # Mô hình này sẽ được copy (mirrored) sang GPU:0, GPU:1, v.v...
    model = tf.keras.models.Sequential([
        tf.keras.layers.Dense(128, activation='relu'),
        tf.keras.layers.Dense(1)
    ])
    model.compile(optimizer='adam', loss='mse')

# Khi gọi model.fit(), TF sẽ tự động:
# 1. Cắt nhỏ Batch dữ liệu ra (ví dụ batch 64 ảnh -> 2 GPU mỗi card lo 32 ảnh)
# 2. Hai card tính toán song song.
# 3. Gom Gradient của 2 card lại, cộng vào, cập nhật bộ trọng số chung, rồi đi tiếp!

Hiểu về bộ nhớ, ổ cứng và PCI-e chính là điều phân biệt một Coder tập sự và một Kỹ sư Hệ thống AI (AI Systems Engineer) thực thụ!
